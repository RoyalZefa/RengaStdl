# Методичка по проекту "LCD панель"

Это самый простой начальный пример из всех учебных проектов.

Он специально сделан очень понятным:

- корпус панели строится одним `CreateBlock(...)`
- экран рисуется как внутренний прямоугольник
- есть один простой `UserEnum`
- есть одно простое использование `SetVisible()`
- нет портов и сложной инженерной логики

## Что моделирует пример

Шаблон описывает простую LCD панель.

У нее есть:

- корпус
- экран
- два варианта установки:
  - настенный
  - настольный

Если выбран настольный вариант, у панели появляется подставка.

## Что изучать в этом проекте

Этот проект лучше всего подходит для самых первых шагов, потому что здесь можно понять базовую цепочку:

1. параметр объявлен в `parameters.json`
2. параметр прочитан в `main.lua`
3. параметр влияет на геометрию
4. результат видно в Renga

## Разбор parameters.json

Откройте `02-lcd_panel/parameters.json`.

### metadata

Здесь задаются:

- имя стиля
- описание
- версия
- автор

Это паспорт шаблона.

### Dimensions

Здесь объявлены размеры:

- `PanelWidth`
- `PanelHeight`
- `PanelDepth`
- `ScreenMargin`

Они задают:

- габариты корпуса
- толщину панели
- отступ экрана от края

### Mounting

Здесь есть:

- `MountType`
- `StandDepth`

`MountType` - это `UserEnum`, который переключает поведение модели:

- `Wall` - настенный монтаж
- `Table` - настольный монтаж

`StandDepth` нужен только для настольного варианта.

## Разбор main.lua

Откройте `02-lcd_panel/main.lua`.

### 1. Чтение параметров

Сначала скрипт получает параметры через:

```lua
local parameters = Style.GetParameterValues()
```

Потом читает размеры панели и отступ экрана.

### 2. Простая логика через UserEnum

Вот ключевая строка:

```lua
local isTableMount = parameters.Mounting.MountType == "Table"
```

Она превращает текстовое значение из `UserEnum` в удобный логический флаг.

### 3. SetVisible()

Скрипт делает так:

```lua
Style.GetParameter("Mounting", "StandDepth"):SetVisible(isTableMount)
```

Это значит:

- если монтаж настольный, параметр `StandDepth` показывается
- если настенный, он скрывается

Это один из самых простых и полезных примеров использования `SetVisible()`.

### 4. Геометрия корпуса

Корпус строится через:

```lua
local body = CreateBlock(panelWidth, panelDepth, panelHeight)
```

Это обычный параллелепипед.

### 5. Геометрия подставки

Если выбран настольный монтаж, создается еще одно тело:

- подставка

Потом она объединяется с корпусом через `Unite(...)`.

Это хороший первый пример условной геометрии:

- один режим монтажа -> одна геометрия
- другой режим монтажа -> геометрия расширяется

### 6. 2D-символика

Функция `createFrontSymbol()` строит:

- внешний прямоугольник панели
- внутренний прямоугольник экрана
- заливку экрана

Это очень наглядный пример 2D-символики.

### 7. SymbolGeometry и SymbolicGeometry

Скрипт задает:

- `Style.SetSymbolGeometry(...)`
- `Style.SetSymbolicGeometry(...)`

То есть и компактный символ, и условную геометрию в модели.

## Практика по шагам

### Упражнение 1. Измените размеры панели

Поменяйте:

- `PanelWidth`
- `PanelHeight`
- `PanelDepth`

После сборки посмотрите, как изменился корпус.

### Упражнение 2. Измените размер экрана

Поменяйте:

- `ScreenMargin`

После сборки посмотрите, как меняется внутренний прямоугольник экрана.

### Упражнение 3. Переключите монтаж

Поменяйте:

- `MountType` с `Wall` на `Table`

И проверьте:

- появляется ли подставка
- показывается ли параметр `StandDepth`

### Упражнение 4. Измените глубину подставки

Если выбран `Table`, поменяйте:

- `StandDepth`

И посмотрите, как меняется форма подставки.

## Сборка

Из корня проекта:

```powershell
.\RengaSTDLSDK\RstBuilder\RstBuilder.exe .\02-lcd_panel\parameters.json .\02-lcd_panel\main.lua -s 1.0 -o .\02-lcd_panel\lcd-panel.rst
```
