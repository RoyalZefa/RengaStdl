# Методичка по Placement3D

Этот документ нужен не просто для справки, а для нормального понимания того, как в `Renga STDL` связаны:

- мировая система координат Renga;
- локальная система координат фигуры;
- точки `Point2D(...)` в профиле;
- точка `Point3D(...)` в `Placement3D(...)`;
- направление `Extrude(...)`.

Главная цель этой методички:

`научиться смотреть на placement и заранее понимать, как будет стоять тело в Renga`

---

## 1. Что вообще происходит в этом учебном примере

В проекте `08-minimal` мы делаем очень простую вещь:

1. берем размеры из `geometry.json`;
2. строим 2D-контур по точкам `Point2D(...)`;
3. превращаем его в объем через `Extrude(...)`;
4. задаем положение и ориентацию этого объема через `Placement3D(...)`.

То есть логика такая:

```text
JSON -> размеры -> 2D-профиль -> Placement3D -> Extrude -> 3D-тело
```

Важно понять:

- `profilePoints` отвечают за форму;
- `Placement3D(...)` отвечает за положение и поворот;
- `Extrude(...)` отвечает за выдавливание профиля в объем.

---

## 2. Мировая система координат Renga

Сначала всегда надо держать в голове мировые оси модели.

В учебных объяснениях мы используем такую схему:

```text
             Z
             ^
             |
             |
             O ---------> X
            /
           /
          Y
```

Проще запомнить так:

- `X` — влево / вправо
- `Y` — вперед / назад
- `Z` — вверх / вниз

Это не система координат одного конкретного объекта.
Это система координат всей сцены Renga.

Именно относительно этих осей вы потом задаете:

- где стоит объект;
- куда он направлен;
- куда он выдавливается;
- как он повернут.

---

## 3. Локальная система координат объекта

Когда вы пишете:

```lua
Placement3D(point, zAxisDirection, xAxisDirection)
```

вы создаете для профиля локальную систему координат.

То есть у объекта появляется:

- свое начало координат;
- своя локальная ось `X`;
- своя локальная ось `Z`;
- и еще одна локальная ось, которая вычисляется автоматически.

Это очень важный момент:

`Placement3D не рисует геометрию сам по себе. Он задает систему координат, в которой будет жить геометрия.`

---

## 4. Официальная сигнатура Placement3D

Для вашего SDK правильная сигнатура такая:

```lua
Placement3D(point, zAxisDirection, xAxisDirection)
```

То есть по порядку:

1. `point`
2. `zAxisDirection`
3. `xAxisDirection`

Это важно, потому что часто в интернете или в чьих-то конспектах можно встретить путаницу с порядком осей.

В вашем локальном `RengaSTDLSDK` документация показывает именно этот порядок:

```text
Placement3D(Point3D point, Vector3D zAxisDirection, Vector3D xAxisDirection)
```

---

## 5. Что значит каждая строка в Placement3D

### 5.1. Первая строка: `Point3D(...)`

Пример:

```lua
Point3D(0, 0, 0)
```

Это точка, в которую ставится начало локальной системы координат профиля.

Важно:

- это не “центр объекта” по умолчанию;
- это не “нижняя точка объекта” по умолчанию;
- это просто начало локальной системы.

Если вы меняете только `Point3D`, форма не меняется.

Меняется только положение объекта в мировой системе координат.

### 5.2. Вторая строка: `zAxisDirection`

Пример:

```lua
Vector3D(0, 1, 0)
```

Это направление локальной оси `Z` placement-а.

В учебном примере это особенно важно, потому что:

`Extrude(...)` идет вдоль этой локальной оси

То есть:

- если здесь `Vector3D(0, 1, 0)`, тело растет по мировой `Y`;
- если здесь `Vector3D(0, 0, 1)`, тело растет по мировой `Z`;
- если здесь `Vector3D(1, 0, 0)`, тело растет по мировой `X`.

### 5.3. Третья строка: `xAxisDirection`

Пример:

```lua
Vector3D(1, 0, 0)
```

Это направление локальной оси `X`.

Эта строка отвечает за то, где у профиля “право”.

Она не задает глубину выдавливания.
Она задает поворот профиля вокруг направления выдавливания.

---

## 6. Что автоматически вычисляется

Когда вы задаете:

- локальную ось `Z`;
- локальную ось `X`;

Renga достраивает третью ось автоматически.

И вот тут обычно начинается путаница у начинающих.

Почему:

- в коде вы задаете только две оси;
- а в реальности объект живет в трехмерном пространстве;
- значит, третья ось обязана появиться автоматически.

Для учебного понимания это можно записать так:

```text
локальная система = origin + локальная Z + локальная X + автоматически вычисленная третья ось
```

Именно из-за этой автоматически вычисленной оси профиль может стоять в Renga не так, как вы сначала ожидали.

---

## 7. Как `Point2D(...)` связан с профилем

В проекте `08-minimal` профиль задается так:

```lua
local profilePoints = {
    Point2D(-60, 0),
    Point2D(60, 0),
    Point2D(60, 40),
    Point2D(30, 100),
    Point2D(-30, 100),
    Point2D(-60, 40),
    Point2D(-60, 0)
}
```

Это еще не 3D-объект.
Это только 2D-контур.

Для учебной записи мы читаем точки так:

```text
Point2D(x, z)
```

То есть:

- первая координата — положение по локальной `X`;
- вторая координата — положение по второй оси плоскости профиля.

На плоской схеме это удобно рисовать как `X-Z`:

```text
Z
^
|                    5 (-30, 100) -------- 4 (30, 100)
|                        /                      \
|                       /                        \
|                      /                          \
|         6 (-60, 40)                               3 (60, 40)
|              |                                         |
|              |                                         |
|              |                                         |
|         1 (-60, 0) ------------------------------ 2 (60, 0)
|
+----------------------------------------------------------------> X
```

Именно на этом этапе большинство людей мыслят так:

- “низ” фигуры — это `z = 0`
- “верх” фигуры — это `z = 100`

На плоской схеме это правильно.

Но после `Placement3D(...)` этот “верх” уже может смотреть куда угодно в мировой системе.

---

## 8. Самая важная мысль про поворот

Нужно разделять две вещи:

1. как профиль нарисован у вас на бумаге или в голове;
2. как этот профиль развернут в мировом 3D-пространстве.

На плоской схеме вы видите:

```text
z растет вверх
```

Но в реальной модели Renga после `Placement3D(...)` это может превратиться в:

```text
вторая координата Point2D уходит в сторону -Z мира
```

И тогда “верх” профиля на схеме станет “низом” объекта в модели.

Именно это вы сейчас и наблюдаете в `08-minimal`.

## 9. Разбор вашего текущего placement

В проекте сейчас стоит:

```lua
local placement = Placement3D(
    Point3D(0, 0, 0),
    Vector3D(0, 1, 0),
    Vector3D(1, 0, 0)
)
```

Разберем по шагам.

### Шаг 1. Начало локальной системы

```lua
Point3D(0, 0, 0)
```

Начало профиля ставится в мировой ноль.

### Шаг 2. Куда идет Extrude

```lua
Vector3D(0, 1, 0)
```

Значит, локальная ось выдавливания направлена вдоль мировой `Y`.

То есть тело растет по мировой `Y`.

### Шаг 3. Где у профиля “право”

```lua
Vector3D(1, 0, 0)
```

Значит, локальная ось `X` профиля направлена вдоль мировой `X`.

### Шаг 4. Что происходит с “высотой” профиля

Вот здесь и появляется важный эффект.

В текущем примере вторая координата `Point2D(x, z)` после placement-а уходит не в `+Z` мира, а в `-Z` мира.

Из-за этого:

- точки с большей второй координатой становятся ниже в мировой модели;
- точки с меньшей второй координатой оказываются выше;
- поэтому широкая часть вашей фигуры визуально смотрит вверх.

Это не ошибка Renga.
Это результат выбранной ориентации локальной системы.

---

## 10. Почему кажется, что объект “перевернут”

Посмотрим на точки:

```lua
Point2D(-60, 0)
Point2D(60, 0)
...
Point2D(30, 100)
Point2D(-30, 100)
```

На плоской схеме:

- основание шире;
- верх уже.

Значит, интуитивно хочется ожидать:

```text
широкая часть снизу
узкая часть сверху
```

Но в текущем placement-е ось “высоты профиля” развернута так, что положительные значения второй координаты уходят вниз по мировому `Z`.

Поэтому в Renga вы видите:

```text
широкая часть выше
узкая часть ниже
```

То есть сам профиль не изменился.
Изменилось только то, как он ориентирован в мировой системе координат.

## 11. Как думать правильно: рабочий алгоритм

Вот самый полезный способ читать любой `Placement3D`.

### Шаг 1. Сначала игнорируйте 3D

Сначала смотрите только на профиль:

- где у него низ;
- где верх;
- где лево;
- где право.

### Шаг 2. Определите направление выдавливания

Смотрите только на вторую строку:

```lua
zAxisDirection
```

И задайте себе вопрос:

`в какую мировую сторону теперь будет расти Extrude?`

### Шаг 3. Определите, где у профиля локальная X

Смотрите только на третью строку:

```lua
xAxisDirection
```

И задайте себе вопрос:

`куда в мировой системе теперь смотрит "право" профиля?`

### Шаг 4. Помните, что оставшаяся ось достраивается автоматически

Именно здесь обычно возникает:

- переворот;
- неожиданный поворот;
- ощущение, что объект “зеркальный” или “лежит не так”.

## 12. Простые сравнения: что меняет каждая строка

### Пример A. Меняем только Point3D

Было:

```lua
Placement3D(
    Point3D(0, 0, 0),
    Vector3D(0, 1, 0),
    Vector3D(1, 0, 0)
)
```

Стало:

```lua
Placement3D(
    Point3D(0, 100, 0),
    Vector3D(0, 1, 0),
    Vector3D(1, 0, 0)
)
```

Что меняется:

- тело не меняет форму;
- тело не меняет поворот;
- тело просто сдвигается по мировой `Y`.

### Пример B. Меняем только `zAxisDirection`

Было:

```lua
Vector3D(0, 1, 0)
```

Стало:

```lua
Vector3D(0, 0, 1)
```

Что меняется:

- профиль остается тем же;
- направление Extrude меняется;
- тело начинает расти вверх, а не вперед.

### Пример C. Меняем только `xAxisDirection`

Было:

```lua
Vector3D(1, 0, 0)
```

Стало:

```lua
Vector3D(0, 0, 1)
```

Что меняется:

- глубина выдавливания может остаться той же;
- но сам профиль разворачивается;
- “право” фигуры начинает смотреть уже в другую сторону.

## 13. Три учебных placement-а, которые нужно запомнить

### Вариант 1. Выдавливание по мировой Y

```lua
Placement3D(
    Point3D(0, 0, 0),
    Vector3D(0, 1, 0),
    Vector3D(1, 0, 0)
)
```

Как читать:

- объект стоит в начале координат;
- профиль выдавливается по `Y`;
- право профиля смотрит по `X`.

### Вариант 2. Выдавливание по мировой Z

```lua
Placement3D(
    Point3D(0, 0, 0),
    Vector3D(0, 0, 1),
    Vector3D(1, 0, 0)
)
```

Как читать:

- объект стоит в начале координат;
- профиль выдавливается вверх по `Z`;
- право профиля по-прежнему смотрит по `X`.

### Вариант 3. Выдавливание по мировой X

```lua
Placement3D(
    Point3D(0, 0, 0),
    Vector3D(1, 0, 0),
    Vector3D(0, 1, 0)
)
```

Как читать:

- объект стоит в начале координат;
- профиль выдавливается по `X`;
- право профиля смотрит по `Y`.

## 14. Что важно для чемпионатной подготовки

На соревновании чаще всего проблема не в том, чтобы написать `Extrude(...)`.

Проблема обычно в другом:

- объект стоит не туда;
- порт смотрит не туда;
- профиль развернут наоборот;
- верх и низ перепутаны;
- символ и 3D-модель ориентированы по-разному.

Поэтому навык номер один:

`уметь мысленно раскладывать placement на мировые и локальные оси`

Если вы это умеете, то гораздо легче:

- исправить ориентацию объекта;
- развернуть порт;
- понять, почему тело “лежит на боку”;
- быстро адаптировать чужой код из sample-проекта.

## 15. Типичные ошибки новичка

### Ошибка 1. Считать, что `Point3D` меняет форму

Нет.
Он меняет только положение локального начала.

### Ошибка 2. Думать, что третья строка задает выдавливание

Нет.
За выдавливание отвечает вторая строка `Placement3D`.

### Ошибка 3. Считать, что профиль всегда смотрит “как на бумаге”

Нет.
После placement-а его плоскость и ориентация могут быть развернуты как угодно.

### Ошибка 4. Игнорировать автоматически вычисляемую третью ось

Именно из-за нее чаще всего и появляются перевороты, которые кажутся “странными”.

---

## 16. Как тренироваться правильно

Самая полезная тренировка для `Placement3D`:

1. Не менять `profilePoints`.
2. Менять только вторую строку placement-а.
3. Пересобирать `rst`.
4. Смотреть, куда пошло выдавливание.

Потом:

1. Зафиксировать вторую строку.
2. Менять только третью строку.
3. Смотреть, как поворачивается сам профиль.

Потом:

1. Зафиксировать оси.
2. Менять только `Point3D`.
3. Смотреть, как тело смещается.

И только после этого переходить к:

- портам;
- символике;
- сложным инженерным объектам.

## 17. Короткая формула для запоминания

Можно запомнить так:

```text
Placement3D(
  где стоит фигура,
  куда идет Extrude,
  где у фигуры "право"
)
```

То есть:

```lua
Placement3D(point, zAxisDirection, xAxisDirection)
```

## 18. Итог по текущему проекту 08-minimal

В `08-minimal` сейчас важно понять четыре вещи:

1. `profilePoints` задают только форму.
2. `Placement3D(...)` задает положение и ориентацию профиля.
3. `Extrude(...)` выдавливает профиль вдоль локальной оси placement-а.
4. Если фигура в Renga стоит “не так”, сначала надо проверять не точки профиля, а оси placement-а.
