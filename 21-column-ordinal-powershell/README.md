# 21-column-ordinal-powershell

Этот пример показывает, как через `PowerShell` и `Renga API` автоматически заполнить свойство:

- `Порядковый номер в группе`

для **колонн** по правилу задания:

- колонны сначала группируются по паре
  - `Тип группы`
  - `Марка группы`
- внутри каждой такой группы нумерация идет отдельно
- в каждой группе счет начинается с `1`

## Почему не формула

Формула в свойствах Renga хорошо подходит, когда значение считается **по данным самого объекта**.

Но в этой задаче нужно:

1. посмотреть на **все** колонны в проекте
2. разделить их на группы
3. внутри каждой группы присвоить последовательные номера

То есть это уже задача уровня:

- обхода объектов проекта
- группировки
- записи свойств

Поэтому здесь удобнее использовать внешний скрипт через `Renga API`.

## Файлы

- `number-columns.ps1` — рабочий скрипт
- `number-columns.commented.ps1` — тот же скрипт с пояснениями почти по каждому блоку

## Что нужно проверить перед запуском

### 1. Имена исходных свойств

По умолчанию скрипт ищет свойства:

- `Тип группы`
- `Марка группы`

Если в вашем проекте они называются иначе, поменяйте параметры запуска:

```powershell
-GroupTypePropertyName "Ваше имя свойства"
-GroupMarkPropertyName "Ваше имя свойства"
```

### 2. Тип объекта "Колонна"

В скрипте по умолчанию стоит GUID типа объекта:

```text
{d9ee2442-e807-42fb-8fe5-9dcfe543035d}
```

Если в вашей версии API для колонны используется другой GUID, подставьте свой через:

```powershell
-ColumnObjectTypeId "{...}"
```

### 3. Марка группы

Если "Марка группы" хранится не как свойство проекта, а как встроенная марка объекта, включите переключатель:

```powershell
-UseObjectMarkForGroupMark
```

## Пример запуска

```powershell
cd .\21-column-ordinal-powershell

.\number-columns.ps1 `
  -ProjectPath "C:\Projects\Columns.rnp" `
  -Visible `
  -SaveProject
```

## Пример с явными именами свойств

```powershell
.\number-columns.ps1 `
  -ProjectPath "C:\Projects\Columns.rnp" `
  -GroupTypePropertyName "Тип группы" `
  -GroupMarkPropertyName "Марка группы" `
  -Visible `
  -SaveProject
```

## Пример, если марка берется из встроенной марки объекта

```powershell
.\number-columns.ps1 `
  -ProjectPath "C:\Projects\Columns.rnp" `
  -GroupTypePropertyName "Тип группы" `
  -UseObjectMarkForGroupMark `
  -Visible `
  -SaveProject
```

## Как работает нумерация

Скрипт делает так:

1. открывает проект Renga через COM
2. находит все объекты типа `Колонна`
3. для каждой колонны читает:
   - `Тип группы`
   - `Марка группы`
4. собирает ключ:

```text
Тип группы | Марка группы
```

5. группирует колонны по этому ключу
6. внутри каждой группы сортирует объекты по `Id`
7. записывает номера:

```text
1, 2, 3, ...
```

## Что можно адаптировать

- имя итогового свойства:
  - `-OrdinalPropertyName`
- GUID итогового свойства:
  - `-OrdinalPropertyId`
- GUID типа объекта:
  - `-ColumnObjectTypeId`
- источник "Марки группы":
- отдельное свойство
- или встроенная марка объекта

## Важно про дубли свойств

В реальном проекте Renga могут существовать несколько свойств с одинаковыми именами:

- `Тип группы`
- `Марка группы`

но с разными GUID.

В таком случае искать свойство только по имени опасно: можно попасть не в то свойство.

Именно поэтому в скрипте по умолчанию уже прописаны GUID свойств, которые при проверке оказались
реально назначены колоннам в текущем проекте:

- `Тип группы`:
  `{44F93574-95B5-4CAE-90BD-3599AC910A8E}`
- `Марка группы`:
  `{BE8B433A-EE51-49DE-8189-5F6476783E22}`

Если в другом проекте GUID будут иными, подставьте свои через:

```powershell
-GroupTypePropertyId "{...}"
-GroupMarkPropertyId "{...}"
```
