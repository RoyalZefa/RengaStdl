# Методичка по проекту "Пожарный датчик дыма"

Этот проект сделан специально как "сводный" учебный пример.

Он очень простой по форме:

- основание
- цилиндрический корпус
- маленький индикатор

Но при этом в нем собраны почти все важные особенности, которые мы уже разбирали:

- блок `metadata`
- группы параметров
- `Length`, `UserEnum`, `Id`
- чтение параметров в Lua
- создание 3D-геометрии
- создание 2D-символики
- электрические порты
- `Placement3D`
- скрытие и показ параметров через `SetVisible()`

## Что моделирует пример

Шаблон описывает простой автономный дымовой пожарный датчик.

У него есть:

- круглое монтажное основание
- цилиндрический корпус
- встроенный или выносной индикатор
- порт питания
- порт шлейфа сигнализации

## Почему этот пример полезен

Он специально сделан проще, чем реальные инженерные устройства:

- нет сложного профиля
- нет трубных портов
- нет большого количества условий

Но в нем уже есть все основные принципы STDL, которые потом повторяются в более сложных проектах.

## Разбор parameters.json

Откройте `03-smoke_detector/parameters.json`.

### 1. metadata

Здесь задаются:

- имя стиля
- описание
- версия
- автор

Это паспорт шаблона.

### 2. General

В группе `General` есть параметр `Material`.

Он показывает, как в STDL объявляется ссылка на материал Renga через тип `Id`.

### 3. Dimensions

Здесь задаются размеры:

- `BodyDiameter`
- `BodyHeight`
- `BaseDiameter`
- `BaseHeight`
- `CableLength`

Это геометрическая основа модели.

### 4. Mounting

Эта группа показывает работу `UserEnum`.

Есть:

- `MountType`
- `CeilingOffset`
- `WallOffset`

Идея:

- если монтаж потолочный, нужен `CeilingOffset`
- если настенный, нужен `WallOffset`

В Lua один из этих параметров будет скрываться.

### 5. Indicator

Эта группа добавляет еще один пример `UserEnum`.

Есть:

- `IndicatorMode`
- `IndicatorDiameter`
- `RemoteIndicatorCable`

Если индикатор встроенный, нужен его диаметр.
Если выносной, нужен кабель выносного индикатора.

Опять же, лишний параметр будет скрываться через `SetVisible()`.

### 6. Electric

Параметры электрических подключений:

- `PortLayout`
- `TerminalOffset`

Они управляют расположением двух электрических портов.

### 7. ports

У объекта два порта:

- `PowerSupply`
- `AlarmLoop`

Это показывает, как в STDL объявлять электрические подключения.

## Разбор main.lua

Откройте `03-smoke_detector/main.lua`.

### 1. Чтение параметров

Скрипт получает параметры через:

```lua
local parameters = Style.GetParameterValues()
```

Потом достает размеры, режим монтажа, режим индикатора и настройки портов.

### 2. Подготовка логики

Скрипт заранее вычисляет:

- радиусы корпуса и основания
- флаги `isCeilingMount` и `isBuiltInIndicator`

Это удобный прием: сложные проверки потом сводятся к простым логическим переменным.

### 3. SetVisible()

Одна из самых важных частей примера:

- скрывается `CeilingOffset`, если монтаж не потолочный
- скрывается `WallOffset`, если монтаж не настенный
- скрывается `IndicatorDiameter`, если индикатор выносной
- скрывается `RemoteIndicatorCable`, если индикатор встроенный

Это показывает, что STDL управляет не только формой, но и удобством интерфейса.

### 4. 3D-геометрия

Геометрия здесь очень простая:

- основание через `CreateRightCircularCylinder`
- корпус через `CreateRightCircularCylinder`
- индикатор через еще один маленький цилиндр

Потом тела объединяются через `Unite(...)`.

Если индикатор встроенный, он входит в геометрию.
Если выносной, корпус строится без него.

### 5. SymbolGeometry и SymbolicGeometry

Скрипт строит символ датчика в виде:

- внешней окружности
- внутренней окружности
- двух пересекающихся линий

Это хороший пример простой, но читаемой условной графики.

### 6. Порты

У датчика два электрических порта:

- питание
- шлейф сигнализации

Они размещаются через `Placement3D`, а потом сдвигаются вдоль оси через `shiftPlacementByZ(...)`.

Это тот же принцип, что и с трубными патрубками, только без `SetPipeParameters(...)`.

## Практика по шагам

### Упражнение 1. Измените размеры

Поменяйте:

- `BodyDiameter`
- `BodyHeight`
- `BaseDiameter`

Потом пересоберите шаблон и посмотрите, как изменился корпус.

### Упражнение 2. Переключите тип монтажа

Поменяйте `MountType`:

- `Ceiling`
- `Wall`

И посмотрите:

- какой параметр скрывается
- как меняется высотное положение модели

### Упражнение 3. Переключите тип индикатора

Поменяйте `IndicatorMode`:

- `BuiltIn`
- `Remote`

И посмотрите:

- как скрываются параметры
- исчезает ли встроенный индикатор из геометрии

### Упражнение 4. Поиграйте с расположением портов

Меняйте:

- `PortLayout`
- `TerminalOffset`

И смотрите, как переставляются электрические порты.

## Сборка

Из корня проекта:

```powershell
.\RengaSTDLSDK\RstBuilder\RstBuilder.exe .\03-smoke_detector\parameters.json .\03-smoke_detector\main.lua -s 1.0 -o .\03-smoke_detector\smoke-detector.rst
```

## Итог

Если вы поймете этот проект, у вас в голове сложится хорошая "карта" STDL:

- параметры описываются в JSON
- логика живет в Lua
- геометрия строится из примитивов
- 2D и 3D можно задавать отдельно
- порты размещаются через `Placement3D`
- интерфейс параметров можно делать умным
