# Методичка по проекту "Нагревательный бак для воды"

В этой папке находится учебный проект STDL, который моделирует простой накопительный водонагревательный бак.

Файлы проекта:

- `04-water_heating_tank/parameters.json`
- `04-water_heating_tank/main.lua`
- `04-water_heating_tank/README.md`

Это хороший учебный пример, потому что в нем есть сразу несколько важных вещей:

- цилиндрическая 3D-геометрия
- 2D-символика
- два трубных порта
- один электрический порт
- логика скрытия лишних параметров

Пример намеренно проще, чем `05-official_primer`, но уже достаточно реалистичен, чтобы почувствовать, как строится инженерный объект.

## Что делает этот шаблон

Шаблон описывает вертикальный бак:

- цилиндрический корпус
- нижний патрубок холодной воды
- верхний патрубок горячей воды
- электрический вывод питания

Объект параметрический. Это значит, что вы можете менять:

- диаметр бака
- высоту бака
- длину патрубков
- высоту холодного и горячего патрубков
- тип соединения
- сторону расположения патрубков
- сторону и высоту электрического вывода

## Как изучать этот проект

Лучший порядок такой:

1. прочитать `parameters.json`
2. открыть `main.lua`
3. пройти блок геометрии
4. пройти блок символики
5. пройти блок настройки портов
6. потом повторить эксперименты в Renga

## Часть 1. Разбираем parameters.json

Откройте `04-water_heating_tank/parameters.json`.

Структура привычная:

```json
{
  "metadata": { ... },
  "styleParameters": [ ... ],
  "ports": [ ... ]
}
```

### 1. Блок metadata

Он задает:

- имя стиля
- описание
- версию
- автора

Здесь особенно важно поле `defaultName`, потому что именно его вы увидите в Renga после импорта шаблона.

### 2. Группа Dimensions

Это главная группа геометрии.

В ней лежат параметры:

- `TankDiameter`
- `TankHeight`
- `NozzleLength`
- `ColdWaterLevel`
- `HotWaterLevel`

Как они работают:

- `TankDiameter` задает диаметр цилиндра
- `TankHeight` задает высоту бака
- `NozzleLength` задает длину обоих патрубков
- `ColdWaterLevel` задает высоту ввода холодной воды
- `HotWaterLevel` задает высоту вывода горячей воды

Очень важно понимать, что `ColdWaterLevel` и `HotWaterLevel` задаются не в абсолютных мировых координатах, а как параметры внутри локальной модели бака.

### 3. Группа WaterConnections

Здесь собраны параметры трубных подключений:

- `ConnectionType`
- `ThreadSize`
- `NominalDiameter`
- `PortsSide`

Логика такая:

- `ConnectionType` определяет тип соединения
- `ThreadSize` используется для резьбового подключения
- `NominalDiameter` используется для обычного диаметра
- `PortsSide` задает, справа или слева находятся оба водяных патрубка

Это упрощение специально сделано для обучения. Вместо отдельной стороны для каждого порта у нас одна сторона для всей водяной группы.

### 4. Группа Electric

Здесь лежат параметры электрического подключения:

- `CableSide`
- `CableHeight`

Они определяют:

- с какой стороны выходит электрическое питание
- на какой высоте находится точка подключения

### 5. Блок ports

У объекта есть три порта:

- `ColdWaterInlet`
- `HotWaterOutlet`
- `PowerSupply`

Первые два относятся к трубным системам, третий к электрической.

Обратите внимание на важную разницу:

- `styleParameters` описывает, что пользователь настраивает
- `ports` описывает, какие порты вообще существуют у объекта

## Часть 2. Разбираем main.lua

Теперь откройте `04-water_heating_tank/main.lua`.

Разберем его блоками.

## Блок 1. Чтение параметров

В самом начале скрипт получает параметры:

```lua
local parameters = Style.GetParameterValues()
```

Потом достает из них нужные величины:

- диаметр
- высоту
- длину патрубков
- уровни подключения

Это стандартный первый шаг в STDL.

Зачем это делается:

чтобы дальше код работал не с длинными выражениями вроде `parameters.Dimensions.TankHeight`, а с короткими переменными вроде `tankHeight`.

## Блок 2. Расчет общих величин

Дальше вычисляются:

- радиус бака
- сторона патрубков
- сторона электрического вывода
- направления по оси X

Особенно важны две переменные:

- `nozzleDirection`
- `cableDirection`

Если значение равно `1`, объект смотрит вправо.
Если `-1`, влево.

Это очень типичный прием в параметрических скриптах: сводить пользовательский выбор к простому коэффициенту `1` или `-1`.

## Блок 3. Функция getPipePlacement

Эта функция строит `Placement3D` для трубного патрубка.

Она получает:

- высоту патрубка

И возвращает:

- точку вставки
- направление оси порта
- дополнительную ось ориентации

Почему это важно:

порт в Renga определяется не только точкой, но и ориентацией.

То есть мало сказать "патрубок здесь". Нужно еще сказать, в какую сторону он направлен.

## Блок 4. Размещение водяных и электрического подключений

После функции создаются:

- `coldWaterPlacement`
- `hotWaterPlacement`
- `powerPlacement`

Здесь уже видно реальную логику модели:

- холодный патрубок ставится на своей высоте
- горячий патрубок ставится на своей высоте
- электрический вывод ставится отдельно, со своей стороны и высотой

## Блок 5. Скрытие лишних параметров

Функция `hideIrrelevantPipePortParams(...)` делает интерфейс удобнее.

Если выбрано резьбовое соединение:

- показывается `ThreadSize`
- скрывается `NominalDiameter`

Если выбрано не резьбовое соединение:

- показывается `NominalDiameter`
- скрывается `ThreadSize`

Это очень важный прием.

Хороший STDL-шаблон не только строит геометрию, но и делает диалог параметров понятным для пользователя.

## Блок 6. Создание 3D-геометрии корпуса

Корпус создается так:

```lua
local tankBody = CreateRightCircularCylinder(tankRadius, tankHeight)
    :Shift(0, 0, -tankHeight / 2)
```

Что здесь происходит:

1. создается цилиндр
2. потом он сдвигается вниз на половину своей высоты

Зачем нужен `Shift`:

чтобы центр объекта оказался в удобной локальной системе координат, а верх и низ корпуса располагались симметрично относительно точки вставки по высоте.

## Блок 7. Создание патрубков как геометрии

Дальше создается заготовка:

```lua
local nozzlePrototype = CreateRightCircularCylinder(nozzleRadius, nozzleLength)
```

Потом она клонируется дважды:

- для холодной воды
- для горячей воды

И каждый экземпляр трансформируется своей матрицей размещения.

Это важный инженерный прием:

не строить каждый патрубок с нуля, а создать один прототип и использовать его несколько раз.

## Блок 8. Объединение геометрии

После этого все тела объединяются:

```lua
local solid = Unite({tankBody, coldNozzle, hotNozzle})
```

То есть в один итоговый объект собираются:

- бак
- холодный патрубок
- горячий патрубок

Потом эта геометрия передается в `Style.SetDetailedGeometry(...)`.

Это и есть основная 3D-модель объекта.

## Блок 9. Создание символики

Для символики используется функция `createTopSymbol()`.

Она строит:

- окружность
- заливку
- вертикальную линию
- короткую горизонтальную линию

На плане это выглядит как простой условный знак бака.

Здесь есть два уровня:

- `Style.SetSymbolGeometry(...)` задает компактный символ
- `Style.SetSymbolicGeometry(...)` задает условную геометрию объекта в модели

Это хороший пример того, что 2D-символ не обязан повторять 3D-геометрию один в один.

## Блок 10. Настройка портов

Функция `shiftPlacementByZ(...)` нужна, чтобы вынести порт от основания патрубка к его торцу.

Почему это важно:

если поставить порт в начале цилиндра-патрубка, он окажется у корпуса.
А нам нужно, чтобы точка подключения была на конце патрубка.

Функция `setPipeParameters(...)` задает параметры трубного порта:

- либо по `ThreadSize`
- либо по `NominalDiameter`

После этого настраиваются:

- `ColdWaterInlet`
- `HotWaterOutlet`
- `PowerSupply`

Для трубных портов задаются:

- положение
- тип соединения
- размер

Для электрического порта задается только положение.

## Полная логика проекта

Если собрать все вместе, шаблон работает так:

1. читает параметры
2. рассчитывает положения подключений
3. строит цилиндрический корпус
4. строит два патрубка
5. объединяет их в одно тело
6. строит 2D-символ
7. размещает трубные и электрический порты
8. скрывает неактуальные параметры

Это очень хороший "средний" учебный пример: он уже реалистичный, но не перегруженный.

## Практика по шагам

Вот лучший порядок упражнений.

### Упражнение 1. Измените габариты бака

В `04-water_heating_tank/parameters.json`:

- поменяйте `TankDiameter`
- поменяйте `TankHeight`

Потом:

1. соберите `RST`
2. импортируйте его в Renga
3. разместите объект
4. сравните новую форму со старой

Что вы увидите:

- бак станет выше или ниже
- диаметр корпуса изменится
- символика сверху тоже изменится

### Упражнение 2. Переставьте патрубки выше или ниже

Поменяйте:

- `ColdWaterLevel`
- `HotWaterLevel`

После пересборки проверьте, как переместились патрубки.

Это лучший способ почувствовать связь между параметром и геометрическим размещением.

### Упражнение 3. Переключите сторону патрубков

Поменяйте:

- `PortsSide` с `right` на `left`

После пересборки посмотрите:

- как оба патрубка переехали на другую сторону бака
- как изменилась ориентация трубных портов

### Упражнение 4. Поиграйте с типом соединения

В параметрах объекта попробуйте:

- выбрать резьбовое соединение
- потом другое соединение

Смотрите, как скрываются и показываются:

- `ThreadSize`
- `NominalDiameter`

Это закрепит понимание `SetVisible(...)`.

### Упражнение 5. Переставьте электрический вывод

Поменяйте:

- `CableSide`
- `CableHeight`

После этого проверьте положение порта `PowerSupply`.

### Упражнение 6. Измените символ бака

В `04-water_heating_tank/main.lua` внутри `createTopSymbol()` попробуйте:

- убрать короткую горизонтальную линию
- добавить еще одну окружность меньшего радиуса

Так вы сразу почувствуете разницу между 3D-моделью и условным 2D-представлением.


```powershell
.\RengaSTDLSDK\RstBuilder\RstBuilder.exe .\04-water_heating_tank\parameters.json .\04-water_heating_tank\main.lua -s 1.0
```
