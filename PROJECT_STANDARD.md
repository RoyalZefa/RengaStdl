# Project Standard

Желательный стандарт для каждой учебной папки проекта:

- `parameters.json`
  Рабочий файл параметров.

- `parameters.commented.jsonc`
  Учебная версия с комментариями.

- `main.lua`
  Рабочий Lua-скрипт.

- `main.commented.lua`
  Учебная версия с подробными комментариями.

- `README.md`
  Краткое описание проекта и команда сборки.

- `metodichka.md`
  Подробное объяснение проекта.

- `practice.md`
  Практические задания от простого к сложному.

- `build.ps1`
  Локальная команда сборки.

- `*.rst`
  Уже собранный результат, если проект проверен builder-ом.

## Правила именования

Чтобы все проекты выглядели одинаково:

- использовать `parameters.json`, а не разные варианты вроде `configuration.json`
- использовать `main.lua`, а не разные варианты вроде `run.lua`
- использовать `metodichka.md`, а не разные названия уроков
- использовать `parameters.commented.jsonc` и `main.commented.lua` для учебных версий

## Правила ссылок в markdown

- внутри учебных файлов использовать относительные пути
- не вшивать абсолютные пути к папке пользователя
- системные пути оставлять только там, где они реально нужны, например для логов
